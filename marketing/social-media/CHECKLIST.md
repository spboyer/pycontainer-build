# Social Media Launch Checklist

Use this checklist to track your progress through the launch process.

## Pre-Launch Preparation

- [ ] Review and customize LinkedIn post (`linkedin-post.md`)
- [ ] Review and customize X/Twitter post (`twitter-post.md`)
- [ ] Create visual assets:
  - [ ] Terminal screenshot of `pycontainer build` command
  - [ ] Workflow comparison diagram (Docker vs pycontainer)
  - [ ] Optional: Demo GIF or video
- [ ] Test pycontainer-build yourself to ensure it works
- [ ] Update repository README if needed
- [ ] Set up GitHub Issue labels:
  - [ ] `feedback`
  - [ ] `feature-request`
  - [ ] `bug`
  - [ ] `question`
  - [ ] `enhancement`
  - [ ] `good-first-issue`
  - [ ] `help-wanted`
- [ ] Prepare common Q&A responses (see `launch-guide.md`)

## Launch Day (Tuesday or Wednesday recommended)

### Morning (9-10 AM)

- [ ] Post to LinkedIn
  - [ ] Copy content from `linkedin-post.md`
  - [ ] Add visual asset
  - [ ] Double-check all links
  - [ ] Post and pin to profile (optional)
- [ ] Monitor LinkedIn for first hour
- [ ] Respond to any immediate comments

### Afternoon (1-2 PM)

- [ ] Post to X/Twitter
  - [ ] Copy content from `twitter-post.md`
  - [ ] Add visual asset
  - [ ] Double-check character count
  - [ ] Post and pin to profile (optional)
- [ ] Monitor Twitter for first hour
- [ ] Respond to any immediate comments

### Evening

- [ ] Check both platforms for comments
- [ ] Respond to all comments received
- [ ] Note any interesting feedback or questions
- [ ] Check GitHub for new stars/issues

## Days 2-3

- [ ] Check platforms 3-4 times per day
- [ ] Respond to all new comments
- [ ] Direct feature requests to GitHub Issues
- [ ] Direct bug reports to GitHub Issues
- [ ] Share interesting discussions

## Week 1

- [ ] Continue monitoring (2-3 times daily)
- [ ] Track metrics:
  - LinkedIn: ___ likes, ___ comments, ___ shares
  - Twitter: ___ likes, ___ retweets, ___ replies
  - GitHub: ___ stars, ___ forks, ___ issues
- [ ] Consider cross-posting to communities:
  - [ ] Reddit (r/Python, r/devops)
  - [ ] Dev.to blog post
  - [ ] Hacker News (Show HN)
  - [ ] Python Discord
  - [ ] Relevant forums

## Week 2

- [ ] Create follow-up post with initial feedback summary
- [ ] Share on LinkedIn
- [ ] Share on Twitter
- [ ] Highlight interesting use cases or contributors

## Month 1

- [ ] Review overall metrics
- [ ] Identify early adopters
- [ ] Consider writing case study blog post
- [ ] Plan next phase of community building

## Community Building (Ongoing)

- [ ] Set up GitHub Discussions
- [ ] Consider Discord/Slack community
- [ ] Write follow-up blog posts
- [ ] Submit talks to meetups/conferences
- [ ] Engage with contributors

## Crisis Management (If Needed)

- [ ] Acknowledge critical issues immediately
- [ ] Create GitHub issues for bugs
- [ ] Provide workarounds when possible
- [ ] Update posts if critical blockers found
- [ ] Stay professional with negative feedback
