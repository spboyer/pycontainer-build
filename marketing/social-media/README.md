# Social Media Announcement Materials

This directory contains ready-to-use social media posts and guidelines for announcing pycontainer-build to the developer community.

## Contents

- **`linkedin-post.md`** - Professional LinkedIn announcement post with posting tips
- **`twitter-post.md`** - Concise X/Twitter post with alternative versions and threading strategy
- **`launch-guide.md`** - Comprehensive guide for scheduling, publishing, and monitoring posts
- **`CHECKLIST.md`** - Launch execution checklist for tracking progress through the campaign

## Quick Start

1. **Review the posts:**
   - Read `linkedin-post.md` for the LinkedIn announcement
   - Read `twitter-post.md` for the X/Twitter announcement
   - Customize messaging as needed

2. **Use the checklist:**
   - Open `CHECKLIST.md` to track your progress
   - Check off items as you complete them
   - Add notes and feedback as you go

3. **Prepare visuals (recommended):**
   - Screenshot of `pycontainer build` command in terminal
   - Diagram comparing Docker workflow vs. pycontainer workflow
   - Optional: Short GIF demo (under 15 seconds)

4. **Schedule and publish:**
   - Follow the timeline in `launch-guide.md`
   - Post to LinkedIn first (morning)
   - Post to X/Twitter 3-4 hours later (afternoon)

5. **Monitor and respond:**
   - Check platforms every 1-2 hours on Day 1
   - Respond to all comments within 2 hours
   - Direct feedback to GitHub Issues

6. **Track metrics:**
   - GitHub stars and issues
   - Social engagement (likes, shares, comments)
   - README views in GitHub Insights
